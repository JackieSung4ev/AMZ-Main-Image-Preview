# GZteam AMZ Main Image Preview

> Amazon search-result mockup tool for comparing your product main image against competitor images on desktop and mobile.

GZteam AMZ Main Image Preview is a Vue + Vite single-page tool for Amazon cross-border ecommerce designers. It simulates an Amazon-style search result page so you can quickly judge whether your main image stands out beside competitor product images.

## Features

- Upload your own main image.
- Pick a saved competitor category gallery.
- Batch upload competitor images for different product categories.
- Drag and drop competitor image folders.
- Switch between desktop, mobile, and dual preview modes.
- Adjust keyword, product count, and your product position.
- Keep realistic desktop product image sizing for visual comparison.
- Export desktop PNG, mobile PNG, current view PNG, or a standalone dual-device HTML file.
- Deploy as a static site with GitHub Pages.

## Tech Stack

- Vue 3
- Vite
- Plain CSS
- GitHub Pages

## How To Use

1. Upload your product main image in the control panel.
2. Pick a competitor category gallery, batch upload competitor images, or use the default competitor image set.
3. Set the search keyword, product count, and your product position.
4. Preview PC, mobile, or dual-device layouts.
5. Export PNG screenshots or a standalone HTML page.

## Competitor Category Gallery

Long-term competitor image libraries live in:

```text
public/competitors/
```

The app reads:

```text
public/competitors/categories.json
```

To add a category later, create a folder such as:

```text
public/competitors/mylar-bag/
  01.jpg
  02.jpg
  03.jpg
```

Then add it to `categories.json`:

```json
{
  "id": "mylar-bag",
  "name": "Mylar Bag",
  "keyword": "mylar bag",
  "path": "competitors/mylar-bag",
  "files": ["01.jpg", "02.jpg", "03.jpg"]
}
```

Use 30-40 competitor main images per category when possible.

## Deploy To GitHub Pages

1. Create a new GitHub repository. Suggested repository name:

```text
gzteam-amz-main-image-preview
```

2. Push this project to GitHub:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USER/gzteam-amz-main-image-preview.git
git push -u origin main
```

3. Open the repository on GitHub.
4. Go to `Settings` -> `Pages`.
5. Under `Build and deployment`, set `Source` to `GitHub Actions`.
6. Go to the `Actions` tab and wait for `Deploy to GitHub Pages` to finish.
7. Open the published Pages URL shown in the workflow or in `Settings` -> `Pages`.

The workflow in `.github/workflows/deploy.yml` will run `npm ci`, `npm run build`, and deploy the generated `dist` folder.

## Custom Domain

This project is configured for:

```text
preview-amz.sudob.com
```

Steps:

1. In GitHub, go to `Settings` -> `Pages`.
2. In `Custom domain`, enter `preview-amz.sudob.com`.
3. Click `Save`.
4. In your DNS provider, add a CNAME record:

```text
Type:  CNAME
Name:  preview-amz
Value: YOUR_USER.github.io
```

5. Wait for DNS to take effect. This can take minutes, and sometimes up to 24 hours.
6. Back in GitHub Pages settings, enable `Enforce HTTPS` after GitHub finishes checking the domain.

The `public/CNAME` file is included so Vite copies the custom domain into the `dist` folder during deployment.

## Notes

- GitHub Pages is public-facing, so do not upload private client assets if the repository or published site should not be public.
- The current Vite config uses relative asset paths, which helps the app work both under a repository path and under a custom domain.
- Keep default app images in `public/assets` so Vite copies them into the production build.
